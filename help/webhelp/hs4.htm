Configuration
configuration
Introduction
BistroMath offers a lot of possibilities which can become confusing for the user. Therefore the configuration is organised in three layers of ini-files which reside in the common application data directory  (C:\Documents and Settings\All Users\Application Data\BistroMath or C:\ProgramData\BistroMath). Storing configurations with free choice of name is locked and must be unlocked with <Ctrl>+<Alt>+<R>.
With wedged fields or likewise asymmetric fields should be verified that both direction and signs conform with the used water phantom software. Especially the GT/Inline/Inplane direction is inconsistent defined amongst different vendors and sometimes even different versions of one vendor.



Advanced Mode
To assure a uncomplicated standard mode almost all settings and menu's are locked and can be unlocked by checking Advanced Mode. However, loading presets can and will set all items regardless the state of Advanced Mode. The Advanced Mode state itself is not configurable. When advanced mode is preferred at startup this should be set in the Advanced Settings tab and stored in a configuration file.
On returning to standard mode the last known configuration and preset will be restored from disk, ignoring the stored Advanced Mode preference.
Note that storing a configuration file with any other name than bistromath.ini is also blocked in standard mode. Therefore you need Advanced Mode to write institute.ini.

Simple Mode
The Simple Mode option hides a lot of features which should be carefully considered during configuration. Verify that this mode is unchecked ("off").



Configuration files
Bistromath does not depend on the Windows registry, but saves it own data in configuration files of the .ini type. In version 4 a lot of names were changed. To keep backward compatibility a conversionlist is used to convert any old elements (BM700_renamed_elements.ini). This file can be removed if there are no ini-files written with previous versions.

Institute.ini
When found this file will be taken as only file for all settings. This is intended as reliable standard configuation for all users in one institute.


BistroMath.ini
When institute.ini is not found, bistromath.ini will be used. Also this file holds the full set of settings for the program. The difference is that this file has no protection and can be saved or loaded by any user.

Presets

Preset files contain a limited range of settings, mainly the menu options that can be checked/unchecked. There are a lot of meaningful combinations however that can be related to some "task". As tasks can change, another preset can be loaded. When a preset is loaded as command line parameter, the name is the file name without exclamation and extension (the file name compare!.ini would give "-preset compare"). 

Configuration file overloading
For all configuration and presets files only the parameters found in the file are subject to any changes. All other settings remain unchanged. This implies that after starting BistroMath with a standard setup like institute.ini, any stripped down configuration can be loaded on top of the basic setup.


Ini-files

The most essential configuration data for daily use such as reference depth and normalisation values ar found in the Configuration tab.
The reference depth is used to normalise depth dose scans. Thus all displayed analysis values are based on this normalisation  ("AbsDepth/AbsValue"). The graphical display of the depth dose curve has its own normalisation ("RelDepth/RelValue"). When combined with a reference file the calculated division will be normalised at the same depth. Although the shape of the division will not change drastically the found minimum and maximum will be different. When using generic beams or reference dosimetry at a specific depth this might be the optimal setup.
When the dosimetrical system is based on a depth beyond dose maximum, all depth dose curves should be evaluated relative to this reference point. The text "PDD" is then changed to "RDD".
The Configuration tab is read-only when not in Advanced Mode.

These values are joined with Dose Conversion parameters. Because there can be different film types this may lead to multiple dose depth definitions for one energy. Starting with version 2.00 all definitions for one energy are kept uniform.



Reference directory
The Reference directory is set at installation but can be changed.
