Concepts
general description 
Reliable results
BistroMath is highly adaptable and configurable to users needs. This might also be a pitfall when multiple users are using the same software for a common basic task.
This can be managed with several strategies:
  • saving all preferences in the predefined, "protected", institute.ini or any other user-defined configuration file when used with command-line parameters,
  • using presets to set for specific tasks, optionally loaded at start with command line parameters,
  • using command line parameters to overrule several critical features.
  • a configurable graphical representation to comply with the water phantom software. 
  • blocking most settings at startup in standard mode.

    Configurable output to screen and file
    All visible analysis results can be configured from this list In the special modes additional output can be written to a file and/or the clipboard.
    Also less used features can be hidden for the user.

    Multiple data input routes for multiple files
    A key concept of BistroMath is the observation of the Windows clipboard for plain text data. This clipboard interception can be switched off/on. Data less or equal to the configurable value MinClipBoardBytes or starting with "BistroMath" are ignored. When recognised as valid data they will be used. Also files can be dropped or opened in the classical way. Multiple files can be dropped or opened when the history list is activated.

    Multiple data formats
    A lot of data formats of all major water phantom suppliers, including some binary formats, are supported.

    Data correction
    Probably not much used, but data can be converted to dose with user defined parameters and/or corrected for background. These parameters are related to different file types. 

    Depth dose analysis
    The correct normalisation position is set according to the user settings. Note that fixed depths might be used here. Dependent on these settings "PDD" values might be presented as "RDD"values. Next (p|r)DD10 and (p|r)DD20 positions are calculated. On top of the objective depth dose at certain levels the graphical normalisation point is relevant when the measurement is compared with a reference. Especially for generic beams the depth of standard dosimetry should be taken for normalisation. Now dose differences in the measurement compared to the reference depth curve will be zero at the (grapical) reference depth in stead of dose maximum. The maximum is fitted with a quadratic fit to evaluate the position and height of the maximum. The same code is used for standard filtering with identical settings.

    Relative curves


    A key concept of BistoMath is the comparison of a measurement with its reference curve. Please read the overview. Reference curves are stored by the user and as default loaded automatically, based on machine name, modality, energy, wedge status, field size and depth. The machine name of the reference can be overruled at load time by using a alias. Temporary references can be used also.
    In the image above a measurement is divided with a symmetry corrected reference profile after profile alignment based on matching because of missing penumbras. Both aligment and symmetry correction are noted in the reference information text. For the same reason all results based on the field borders are red. The flatness of the measurement itself is 2.0% but when compared to the reference the range of differences, the relatve flatness, is limited to 1.2%.

    Profile alignment
    When a reference is loaded, the center of the reference is optionally moved towards the center of the measurement. This can be problematic when a field border is missing in measurement and/or reference. Then fallback strategies are used, based on the first derivative or matching as last resort.

    Symmetry correction
    Also, the reference profile might be slightly asymmetrical, which optionally can be corrected. The slope of a straight line, fitted on the data with the In-Field area is applied to the data, starting at the field center.

    Comparison
    The result of the comparison with a reference curve, the calculated curve, is user defined. The primary design target was to calculate the difference as measured/reference, normalised to 100% in the normalisation point. In that case the flatness of the calculated curve also can be evaluated: the relative flatness.
    Alternatively a gamma calculation can be applied.
    The reference may also be used to combine it with the reference to one summed curve. This can simulate the effect of abutted fields.

    Division of measurement and reference
    When the comparison is chosen to be the division of measurement and reference the calculation starts with the raw data of both measurement and reference. After quadratic filtering to temporary buffers the center of both profiles is calculated. Then the divison is calculated on points with the same relative position to the center in their data set. The result curve is passed through a median filter.

    Field types
    Profiles are classified as Standard, FFF, Wedge or Small. This concept is further explained on the Field types page. All related parameers can be found on the Field Types tab.

    Multiple-data files
    Files with several measurements in one single file come in two variants:
  • A series of profiles with no predefined ordening. These are mostly data sets from water phantom like the IBA RFA300 format (*.asc), OmniPro 6 binary (*.rbf), PTW (*.mcc). Any profile in such a set is treated as individual profile.
  • A fixed, organised data set, mostly from a specific detector like a 2D array. The data can be used as indivdual profiles, but the set as such can be stored as reference in the original format. This makes it possible to automatically extract the appropriate curve as reference. 
    For multiple-data files the <PageUP>/<PageDown> keys are used to browse through the file.

    Temporary references
    Any scan can be used as temporary reference, which can be handy to compare two variants of beam settings/machines/fields. Also a check on the scan type can be switched off. When a temporary reference is active, no other reference will be loaded automatically.
