version 4.05.3
build 720, 30/10/2020
  • Small code improvement: The evaluation type "u" which calls for the border postiton at users value, in the SpecialMode2-configuration can now be extended with any numeric value like "unn" where nn will be replaced with the used dose level.
  • Efficiency improvement: All raw data are first checked as potentially binary file type and, when not found to be of a known binary type, opened again in text mode. As remnant from the Delphi era both data streams were kept in memory. Now the binary data stream is copied in memory as raw bytes to a text stream. Then the binary stream is emptied, thus also avoiding to open the file twice.
  • Performance/bug repair: When in the Measurement menu User Dose Level was selected as border evaluation level (and a reference was loaded and alignemnet of the reference was chosen), this was not calculated in due time, which sent BistroMath executing an extremely time consuming profile matching procedure, which might also give a subtle different shift result. Alse the field width at user's level failed to present although both borders were available. Now the shortcut of not calculating this value as default during profile analysis is removed. If not used, the user might configure the User Dose Level to 50%, which is also the default. Then the code will take this already obtained result for maximum efficiency.
  • A minor inconsistency was found: The Linac Error calculation still used a fixed d50 border instead of the border definition of the actual Field Type.
  • Some options, like Detect diagonals and Ignore zero steps were not passed timely to the actually used analysis object (also with no history list), failing some profiles to open.
  • For OmniPro v8 files the by OmniPro added text " Accelerator" was not chopped off, failing to identify a reference.
  • In an attempt to move to standard FreePascal the axis sign correction data were accidently initialised to 0 and not to 1. This might nullify positions also and therefore failing to read the data. At a second attempt this would be repaired.
  • A recently introduced feature to chop off zero data points at the end of files (for Varian Eclipse data points outside the body contour) is now limited to depth doses. This intoduced a failure in the selftest.
  • Files of the w2CAD type failed to read (also found in the selftest).
  • The selftest also failed unexpectedly on the edge detection of wedge profiles. This was tracked down to the derivative function where a valid result was set to invalid. This error was introduced between 2016 and 2018 and also found in the last Delphi releases.
  • For a window larger then the minimum required height BistroMath would decrease the height due to a complete rebuild of the results panel every time another tab was selected. Now this instruction, introduced for good measure, is removed again from the tab switching code.
  • When application of a reference fails, more details will be given if the loglevel is increased.
  • Insertion of the TG line origin in other scan lines for the StarCheckMaxi will be allowed only in DOSE mode.


    version 4.04
    build 714, 16/10/2020
  • Bugfix for StarChck files: Although not complety diagnosed, the combination of using the history list and opening a PTW StarCheck file might not present the inline profile initially and therefore not refresh the stored value for the central detector. When instead the crossline was shown first and the setting Assign origin to both 2D array lines was checked, a wrong value for the central detector was applied. Now it is forced that always the first profile of a multi scan file is presented.
  • New: A Field Type Electrons is introduced.
  • New: Legacy support for oddly formatted numbers.
  • Small code improvement: When not frozen, for data sets a complete reprocess is forced upon cycling through the history list.
  • Field type name in axis title is not converted to lowercase and brackets are removed.
  • Corrected alignment of some text elements in Advanced Settings tab.
  • Edge fallback strategy still compared d50 and derivative instead primary and fallback method. Forcing makes no sense anymore. Just change the primary method.


    version 4.03
    build 710, 12/10/2020
  • New: file input-only support for the Varian Eclipse TPS output format.
  • New: BistroMath uses now the SoundEx function to find month text values for multiple languages. For example 'may', 'mai' and 'mei' all give the same soundex function result. Therefore, when text is used for the month, the order 'MDY' and 'DMY' are now working irrespective the real order for day and month.
  • Small code improvement: a temporary reference with lacking information parameters could not be passed through the history list as valid reference, even with Check Temporary Scan type switched off.
  • Small code improvement: Eclipse files with multiple zero-value data points ate the end were found. This might give a large overshoot of the used quadratic filter at the very end of a pdd. This had only effects on the maxium of the graph but annoying anyway. Now those points are chopped off in an early stage.


    version 4.02
    build 707, 03/10/2020
  • Important bugfix: Due to a code revision error for the history list, version 4.01 accesses undefined memory locations for SpecialMode2 handling only. This is repaired.
  • Code improvement: the history feature did not combine very well the temporary reference feature. Now the the latest temporary reference is transferred to the next data set (when not frozen). See details. A complete review of the reference loading process was done. It is higly complicated by the multiple types of possible data types in both text and binary format and whether or not a data set is frozen upon cycling through the history list.
  • New: When a temporary reference is set, this will now be applied immediately on the current data set.
  • New: When a temporary reference is taken
  • New: When a large data set is defined as temporary reference, this may take a while. Progress messages will show on the statusbar.
  • New: The sigmoid penumbra model can be plotted over the measurement or reference (if selected). This feature can be actived in the View menu. This and many other grapical elements can be selected.
  • The Indicators item in the View menu is renamed to In-Field Area indicators.
  • When paging through the history list, the last analysis message stays visible although this might disagree with the newly displayed data set. Therefore now a message on the set Field Type is shown.


    version 4.01
    build 704, 19/09/2020
  • In any saved configuration file (not presets) information on SpecialMode items will be inserted.
  • SpecialMode[2]Float[2] (wedge detection) is not used anymore; standard analysis will do this with the given settings on the Field Types tab.
  • The text format of the left axis markers is optimised for both Zoomed and Unzoomed mode.
  • New: A history list is introduced.
  • New: Multiple files can be dropped or opened, limited by the size of the history list.
  • Added some condional compiler statements for the Fedora platform. By definition no effect for the here published Windows platform release.


    version 4.00
    build 699, 14/08/2020
  • All changes, most of them initiated in the porting process, as published in v3.30 and subversions are ported to version 4.00. This is a long list as the last 3.30 subversion was published in july 2020.
  • Due to the deprecated support of the x87 math processor in Windows10/64bits exact reproduction with previous versions before 3.30.18 is not possible.
  • New: The sigmoid model may be used as alternative to calculate 50% border positions. Also the inflection point is more precise calculated as the derivative function which amplifies the noise by its nature.
  • New: The profile value at a position as percentage of the border to center of field is introduced (Profile Evaluation Point) as evaluation type "r" / "R" for the results panel and specialmode2.
  • New: The first page of this helpfile contains a clickable image for fast navigation.
  • New: In the viewing menu an Auto Unzoom item is added for FFF profiles.
  • New: All menu shortcuts can be changed through the configuration file.
  • New: The right axis can be linked to the grid.
  • New: A pdf print of the current window.
  • New: In the Result Panel rules two new conditions (N=Standard field type, S=Small field type, see below) and field size limits are introduced.
  • Changes in configuration file: a lot of elements were renamed. Backward compatibility is guaranteed with on-the-fly conversion. See configuration issues.
  • Minor design change: The statusbar is now split in two parts. At the right side the confirmed scan number in a multi scan file is shown; for a single data set the number of measurement points.
  • Minor design change: The right axis labels are now aligned with the left axis grid at the minor costs of less nice numbers.
  • Minor design change: The SelfTest is removed from standard release builds although the code is maintained and used during development. Therefore SpecialMode4 is also removed.
  • Minor design change: Graph panel, graph grid and general panels (as a group) have configurable colors.
  • Minor design change: When switching between tabs the shift of a curve is preserved; using Load current data in the File menu does not.
  • Design change: Edge annotations which reflect the primary edge definition can be suppressed.
  • Design change: A new information type is introduced: the Field type. This replaces the (unnamed) default, Wedge and FFF charcteristic with the new Field types Standard, FFF, Small and Wedge.
  • Design change: The annotation for edge values based on the sigmoid inflection point is changed from s to inn, where nn is the actual dose level. A new 50%-level based on sigmoid analysis with annotation s is introduced. The evaluation types for these choices are 'i' and 'I' respectively. The sigmoid may also be used to find the position at user's level (evalution type 'U').
  • Minor bug fix: For generic files the measurement option Generic to PDD failed.
  • Minor bug fix: The generic data reading object was too picky on empty lines and non-printable characters.
  • Minor bug fix: On changeover from a single scan to a array scan data set that started with the same scan type, SSD, field size and depth the single scan reference was maintained as "valid". Only when changing to the next scan in the data set the correct reference arrray scan was taken.
  • Small improvements for the histogram presentation to get a better normalisation.
  • A lot of visual elements have been renamed or replaced with other structures completely.

    build 700, 07/09/2020
  • The Settings tab and Advanced Setting tab are slightly reorganised.
  • Bug fix: A reporting message for the second or next reference of a multireference caused an unhandled error.
  • Bug fix: Added aliases were never used.
  • Bug fix: always the FFF radius was used for the Top model.
  • New:The Field Type MRlinac is added.
  • New: The application of the Top model is widened to all Field types.
  • New: Added 'C' als Evaluation Type. to show the curvature of the Top model.
     
  • New: Actually applied aliases for the current measurment are marked in the Aliases tab.
  • New: The inflection point based penumbra width is now supported according to NCS-33. Therefore the functionality of the Results Panel rule 'p' is also changed.

    build 701, 08/09/2020
  • Minor design change: Now autolooping (<Ctrll>+<PgDown>) ends effectively with the last valid scan in a multiscan file.
  • New: Autocentering of a profile is now optional in SpecialMode3.

    build 702, 09/09/2020
  • Minor bug: In the file Conversion tab the field to change the name template did not become visible.
  • New: The .rfb file type is now also supported in the Conversion tab.
  • For multi scan files there will be displayed one line per scan during conversion.
