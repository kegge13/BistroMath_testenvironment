Results panel
configuration
Starting with version 3.20 all results in this panel are completely configurable. The explanation below assumes a standard setup.
The v3.20 rule set has been carefully designed to reflect previous versions. All values are clickable: the result will be copied to the clipboard.

The results of the analysis are shown below the graph and depend on the type of profile:

Inline/crossline profiles

The left and right positions values are presented for a filtered curve according to the chosen definition (50%, user set value, derivative edge detection or sigmoid model). The exception here is the sigmoid model; because of its statistical nature it is based on the unfiltered measurement. For this purpose the main axis is chosen as reference point if the main axis falls within the measuring range and the dose value is at least 95% of the maximum dose. Otherwise first the left and right position are based on the dose maximum and the new normalisation point is chosen as the average of these preliminary positions. This normalisation point can also be set.
The flatness is calculated as defined by NCS-8 within the central 80% of the 50% width. Also the maximum value within this area is given. When a reference file is used to calculate a relative error and the calculated curve is set to visible, also the relative flatness is given. Alternatively, a Gamma analysis can be calculated. Then the Confidence Limit is given as m+1.5•s of the gamma curve (where m and s are the estimated mean and standard deviation).


When the origin is chosen as Center of Field and for FFF profiles the In-Field field CoF strategy is taken as default strategy, not the center will be reported but the position of the top. There are two different models available: the Top model and the Slopes model. Also, the penumbra values are replaced with the slope values (in %/cm) as found with the sigmoid model. Note that the slope is much more sensitive to the quality of the fit and the chosen fit range.

The colors are meaningful:

The color of the curve titles (i.e. equipment and date) corresponds with the curves in the graph.
Green values and labels: the curve shifted. The measurement may be shifted towards the center by the user. As standard behavior the reference curve is aligned with the measurement. The needed shift for the reference profile is reported in the curve title.
Blue values: non-standard results on corrected curves (i.e. symmetry correction on measurement curve).
Red values: non-standard analysis results.

Depth dose curves

The PDD is normalised on a reference depth if this is given. Otherwise the dose maximum is used. See Configuration for a detailed explanation with examples.

Annotations
There are several annotations for the center position, profile width and field borders:
 (c): Based on the calculated center, the field is moved at the origin.
 (b): Standard 50% border detection is used to calculate the field borders; not displayed for center position.
 (e): Edge detection is used to calculate the field borders.
 (i): Inflection point with the sigmoid model is used to calculate the field borders.
 (s)  The 50% level is derived from the sigmoid model.
 (nn): The user dose level at nn% is used for the field borders.
 (T): The Top model is used to find the top of a FFF field.
 (S): The Slopes are used the top of a FFF field.
For values dependent on the normalisation:
 (n): The normalisation is different from the standard (Normalisation on center).
 (*) A configured normalisation value and/or depth is applied for a depth dose curve.
There are annotations for flatness and symmetry:
 (s): The measurement is corrected for symmetry (see also example below).
 (F) A FFF-profile is detected. The flatness is given the slope in percent per cm.
General annotations:
 (z) The SDD of a profile is changed.
 (f) Fitted data are used for depth dose analysis.
 (r) Data are resampled.
 (~) The result is changed by a manual shift.
 (!) The result is based on a synthetic profile.

Curve information
Curves can be labeled as "composite" when it is not a straightforward (filtered) measurement anymore. This is true for merged data, derivative, gamma calculation and the ratio of two curves. 
For the Measurement and Reference curve a composite curve this shown as "Composite(curve information)".
Filtering and symmetry correction are not denoted as composite results but have their ow explicit information added.

Curve selection
When the coloured name/date label is clicked, this will select the corresponding curve in the graph area will be selected.
