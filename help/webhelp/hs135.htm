Reference menu

reference menu 


The reference menu regulates the core features of Bistromath. All items in this menu can be (un)checked or to (in)activate the option. For most options the current profile is automatically reloaded to reflect the changes.

Automatic load of Reference
When checked the relevant reference profile is loaded and presented as the Reference curve.

Device specific Reference
This option is intended for selected 2D-arrays with multiple curves with a fixed order in one single file. When applicable a reference may be stored in the original file format. The appropriate curve will always be taken from the reference file.
At this moment only the PTW Starcheck Maxi and Sun Nulear IC PROFILER are supported. Other devices may be added on request. This option needs specific information in the configuration file to work properly. When this information is missing, this menu item will be disabled.

Reference from Generic Beam List
With this option switched on, Generic Beam list becomes available in the Confiuration tab. Both this setting and the list can be stored in any configuration file.

Index multi reference
With this option switched on, much more possibilities are opened by making an index of all the scans in a reference file. Of course, this may take sisgnificant time as all these scans have to be analysed. Now different file formats for the PTW Starcheck can be mixed and unsorted multi reference files can be opened as temporarary reference. Note that any entry in a multiple scan file that cannot be analysed as qualified scan is simply ignored. Indexing large files might take considerable time however. When multiple scans qualify as reference, the first one will be taken.

Reference at default SSD
When a measurement at some SSD is rescaled to the default SSD, this option allows to take automatically the appropriate reference at the default SSD. Note that this option is deactivated (grayed out) when the measurement is not rescaled.

Background correction
See the Background correction item for details.

Symmetry correction of Reference
See the Symmetry correction item for details. For the reference it might be used as standard behavior.

Align with measurement
If this item is checked the reference is shifted towards the measurement, based on the difference in center position. As (mostly) there are small errors in the setup of the watertank, the standard choice is to activate this option. When unchecked also differences in block/leaf calibration are much more relevant.

Align on Top for FFF
If this item is checked (and enabled) the reference is shifted towards the measurement, based on the difference in position of the top for a FFF profile using the choosen model to calculate the top.

Normalise reference
With this setting normalisation of the reference curve is forced. Under most circumstances this will have little effects as the file comparison mode uses specific strategies to optimise the graphical overlay of the reference and the division itself will be normalised anyway. With the addition of the reference to the measurement this might be a matter of choice.

Calculation is ...
The result of comparing the measurement with the reference can be
 - a division of measurement and reference,
 - a gamma analysis,
 - the addition of measurement and reference,
 - none of the choices above.
