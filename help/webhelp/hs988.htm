Upgrading from version 3.xx to 4.xx
general description 

Significant changes in configuration files
With the port to the Lazarus/FreePascal platform all code has been reviewed. Large portions of the the user interface were reorganised and/or rebuild. Consequently visual elements were renamed or replaced with other structures. As configuration files use the name of the visual elements for storage, old configuration files, including presets, are not completely reliable. For renamed elements this is corrected automatically with the installed repair file BMnnn_renamed_elements.ini, were nnn is the build number. 

After upgrading
During installation there is the option to repair all configuration files. When this option is used the repair file (see above) can be removed.
If the repair file is found by BistroMath this file will be applied on the in-memory version of loaded configuration files when needed; nothing will be written to disk.
In all cases the user should check that all settings are as expected.

Manually rewriting all configuration files
When all configuration files are openend, checked for their result on all settings and rewritten, these files will be free from redundant data. Rewriting the institute defaults file must be unlocked.
