Processing menu
processing menu 


Concepts
In old versions of BistroMath the editor window was reread and processed when preferences or menu options were changed. This could also be enforced by the File menu option Load current data. For binary files with a single curve the data were converted to a text format and put in the editor for convenience and reload of the current data. For multiple curve binary formats like the rfb format this is not implemented. The Nelder-Mead PDD-fit introduced also the need for a reprocessing feature. This is covered in this menu. Also the Use as Temporary reference option is moved from the File menu to this menu.


Autoscaling
This fundamental feature of BistroMath should normally be switched on. It renormalises data to (mostly) 100 at some point, both on the screen and before doing any comparison. There may be situations where an absolute data set is compared with another and no scaling is wanted. Note that with measurement renormalisation a data set can be made absolute. These renormalised data also can be set as Temporary Reference.
Note: Autoscaling is locked in its on state by default. Unlock it with <Ctrl>+<Alt>+<R>.

Example with autoscaling switched off

In this example:
  • Autoscaling is unlocked
  • Autoscaling is switched off
  • Measurement Norm adjust mode is switched on 
  • The Calculation result is set to Gamma analysis
  • A measured pdd is loaded
  • This pdd is normalised to give the expected dose at the reference point
  • The normalised measurement is set as Temporary Reference
  • The calculated pdd is loaded as measurement

    Apply sigmoid model on Buffer
    Normally the buffer contains the derivative of a profile to show the results of the edge detection algorithm. The next level of edge detection is the inflection point of a sigmoid fit. To show the performance of this strategy, all the used data points are replace with the sigmoid fit. Typically this will be visible in the shoulder area where the fit starts to deviate from the data. The penumbra itself should closely follow the data. Note that the inflection point still might become less reliable when applied on conventional profiles where the penumbra becomes an almost straight line.

    Reprocess current data
    This will restart all processing on the raw data as read from file or editor window. The PDD-fit will be started with the current parameters for the best fit.

    Reset Nelder-Mead fit
    A restart as above. The PDD-fit will be started with the initial default parameters.

    Merge Source with measurement
    This activates merging of two scans. See the item Merging of scans.

    Set Measurement as Merge source ("stitching")
    This option sets the first part of the profile/depth dose to combined with (merged with, stichted to) a following scan.

    Clear Merge source
    This option frees the Unrelated curve which is used to store the merge source and thus making it available again for other tasks.

    Mirrored measurement as Reference

    When checked, the current profile is copied to the background reference storage. Then this copied profile is mirrored around the Center of Field. Now the Temporary reference item is activated and the new reference loaded as any other temporary reference. The state of this item will not change until switched off again and will therefore work on any next profile. Taking any other temporary reference or clearing the temporary reference will also unset this option. See the separate item for details.

    Make synthetic profile
    This option activates the creation of a virtual profile from a small(er) FFF-field and and a large FFF-field. 

    Set as Temporary reference <Ctrl>+<T>
    When a measurement is loaded this may be used as temporary comparison to all profiles/depth doses as long as this item is checked. The file is saved in memory. This option is not saved in the configuration file and defaults to "unchecked". See more details in the separate topic.
    Clear Temporary reference <Ctrl>+<Shift>+<T>
    The temporary reference buffer is cleared.

    Ignore name of Treatment Unit <Ctrl>+<U>
    For a quick comparison of two linacs the name of the linac should be ignored in the check of the scan type. By ignoring the linac name it is impossible to select automatically the correct reference. Therefore this works only for temporary references. A more general approach offers the Aliases tab. By making one name equal to another cross-referencing is automated.

    Check Temporary Scan type
    When this item is checked the temporary reference is only used when its scan type equals the scan type of the measurement. If unchecked the reference can be compared with any scan.

    SpecialModes
    The SpecialMode entries are only visible in advanced mode. Each state can be (de)activated here. These states are stored in any configuration file when saved. Can also be started from the command line. Note that extra information should be configured in the actually used configuration file.
