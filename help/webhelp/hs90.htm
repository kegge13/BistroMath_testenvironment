History list
general description

When activated in the File menu or the Advanced Settings tab, BistroMath will keep then multiple data sets in memory. These data sets, multiscan files included, are complete data engines with all related parameters. When opening a new file always the actual settings from the user interface are taken. 

The History list is a linear list but behaves circular, <Ctrl>+<PageUp> will move to the previous data; <Ctrl>+<PageDown> will move to a later (or the oldest) data. When a data set from the history is presented (i.e. not the last loaded) then in the window title the 'historic position' and actual list size are shown as --i/n--. When frozen (see below) a asterisk is added. When data have to be pushed out because new data sets are added, the earliest opened will be replaced.
This cycling feature is designed to be completely equivalent to using a single data set: also all buffers and the editor are restored to guarantee all functionality. When a temporary reference is set this will be transferred to the next data set upon cycling. This also works for multi-scan data sets. Whether or not such passed refference data will be used depends on the receiving data set and the settings in the Processing menu.

Use history list
When checked BistroMath will maintain a list of data sets. The first data set in the list will be displayed. All data will be set to (un)frozen when the state is changed to (un)checked. See below for explanation.

length
This sets the length of the history list whether it is used or not. A value of 1 will uncheck Use history list and disable No recalculations on history data. When the displayed data set is removed from the history list by reducing the list lenght, the first data set in the list will be displayed.

Put history data in frozen state
When checked the data sets are not compressed but stay in a "frozen" state. No synchronisation is done between user interface and a frozen data set. Note that loading new data will always unfreeze the current data set. Using <PageUp> or <PageDown> without <Ctrl> to step through multiscan data will also unfreeze that data set. With unfrozen data the reference is loaded again and all necessary calculations must be done again. Also a manual applied shift is lost.
In all cases, when data are unfrozen, the current settings from the user interface will be taken and differences with the state before might occur.
There are both advantages and disadvantages to using the frozen state:
  • Freezing takes much more memory.
  • Frozen data are loaded much faster as no recalculations will be done.
  • Frozen data might not reflect the current settings in the user interface which might also be exploited by loading the same data with different settings.
  • A temporary reference is not passed to a frozen data set.
