Measurement menu
measurement menu 


All items in this menu can be (un)checked to (in)activate the option. For most options the current profile is automatically reloaded to reflect the changes.

Symmetry correction of Measurement
See the Symmetry correction item for details. For the measurement it should stay unchecked mostly.
The Flatness and Symmetry label will get a "s" as annotation.

Move profiles to origin
When checked, for all profiles which are measured at one fixed depth, the Center of Field value will define the shift of the complete profile. The Center of Field is default at the symmetric 50%-values of the filtered profile, but might also be determined by the edge positions or the field maximum. This should not be confused with manually shifting profiles.
The Center, Width, Left and Right label will get a 'c' as annotation.

Mirror profile
The measurement curve is mirrored. Analysis results are given for the mirrored profile.

Resample data
This item must be checked to resample data to a fixed grid, which is defined in the Settings tab.

Mayneord transform
This applies a Mayneord transform to the current photon depth dose curve. The default SSD is the target value for the transformation.

The application is clearly indicated in the lables.



Extended symmetry submenu

One of the items will be displayed below the standard symmetry item in the results panel, all are evaluated every time however. See the explanatory item for details.

User Dose Level
A non-standard level for field border detection may be defined in the Settings tab but will be used only when menu item is checked.

Use PDD-fit results
When checked, the fitted pdd will be used for data analysis.
This option is grayed out when the cpu-intensive pdd-fit is not activated in the Settings Tab. For the used model and backgrounds see Nelder-Mead fit.


Detect FFF


A set of rules is applied to find FFF-profiles. The flatness is the slope in percent per cm.

FFF Center of Field submenu
The center of the field can be derived from either the penumbras (Border/Edge), set to zero (Origin) or taken from the position of the peak (Peak). The center of the field is displayed as "Center" in the results panel. Note that the analysis region (In-Field area) dpends on the center position.

FFF Normalisation submenu
A separate setting for FFF fields, further identical to conventional field sub menu above.

FFF peak submenu
This submenu offers the possibility to calculate the top position with alternative methods. These alternative methods are only applied when a FFF profile is detected.
  • Top model: the top is fitted with a model. The position of the maximum is taken as center position.
  • Slopes: the crossing point of the calculated slopes is taken as center position.
    Note that, as in the image above, for the non-default options the annotation and color for the center change.

    General detection strategy submenu


    Detect diagonals
    With "diagonals" are meant collimator angles of 45+n•90 degrees here for measurements along the main axes. When a value for the field width is given with the measurement data and the actual field size exceeds this value with more than 5%, then it is assumed the profile is a diagonal measurement or the collimator angle deviates more than 20 degrees from the principal directions (0, 90, 180, 270). The In-Field area for these measurements is 70% of the nominal width. When the diagonal-rule is used, the In-Field area indicators will be red in stead of blue.

    Accept bad penumbra's
    When checked any value for the penumbra is displayed, otherwise a configurable limit will be used. When the average of the penumbra's is above this limit, no punumbra information is displayed. You will also get the message "Penumbra too wide".

    Accept missing penumbra's
    When checked a best guess of the In-Field area is made to calculate the symmetry and flatness.

    Accept zero steps
    When checked multiple contiguous data points with the same position/depth are acceptable.

    Generic to PDD
    When a generic, untyped two-column data set, or a free scan is encountered this will be interpreted as PDD when checked.

    Generic to Electron
    When unknown assume electrons when this item is checked, otherwise the default is photons.



    SSD scaling submenu


    Profiles scaled from SDD to SSD
    When checked for all profiles which are measured at one fixed depth, the positions will be scaled back to the distance of the SSD, i.e. the water surface.
    Field positions are denoted with "(r)". Note that only a geometrical scaling is applied.

    Profiles scaled to default SSD
    When checked for all profiles which are measured at some SDD, the positions will be scaled back to the default SSD. The default SSD can be configured in the Settings tab. When also the option Reference at default SSD in the Reference menu is checked, profiles at some SSD can be compared with original profiles at the default SSD as shown above. Field positions are denoted with "(r)".  Note that only a geometrical scaling is applied.


    Signal correction submenu


    Background correction
    See the Background correction item for details.

    OD to Dose conversion
    OD to Dose conversion may be used for several types of files in the Settings tab, but will be used only when menu item is checked.

    Ionisation to Dose conversion
     This menu item cannot be checked to activate it for all following data like other items in the Measurement menu. Its intended use is application by the user (<Shift>+<D>) after the date have been read. When the conversion is succesfull the menu item will be checked to reflect the current state. It is always reset for the next curve. The user is responsible for applying this function to the appropriate data: electron depth dose curves measured with ionisation chambers. There is no way to detect the used detector type in BistroMath. The other two conditions (electrons, depth dose) will be checked however. This function can be applied only once on a data set. Returning to the original data is possible by rereading the data (<Ctrl>+<L>). See also the scientific backgrounds.

    Renormalise source data
    When checked the source data of the measurement will be normalised to the actual normalisation settings. This opens to possibility to save normalised data.

    Axis manipulation submenu


    Preserve original data on export
    This option in the Measurement menu regulates the effects of axis swapping on export. Its default state is on. Switching off this option introduces serious data integrity problems because swapped data will be interpreted as original, unswapped data when opened again from file. This can undermine institution wide conventions for data interpretation. Therefore the state of this menu item is always on at startup and is not taken from any configuration file. Moreover the item is locked at program start.

    Invert axis (axis swapping)
    The direction of the data representation on the screen may be inverted when comparing side by side with the measuring system. The axis designations and/or the axis direction might be wrong. With these inversion options the axis sign (direction) is inverted at file read time and is reflected in the bottom axis labels. This has effect on all relevant analysis results..

    Alternative axis setup
    When, for any reason, the data columns for the coordinates do not meet the format conventions or are registered as the wrong scan type, this can be corrected by specifying the correct order of Inplane, Crossplane and Depth in the advanced settings tab. The feature is only used if Alternative axis setup checked. Axis swapping and axis sign inversion are applied as configured.
