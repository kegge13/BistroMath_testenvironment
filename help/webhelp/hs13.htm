Axis swapping
view menu  | measurement menu

For most users it is prefered that the graph view matches with the profiles as seen in the water phantom software. Notably the GT-axis is handled inconsitent by different vendors or even different versions of one vendor. Therefore here the GT (inline/inplane) axis is taken as example. The axis designations and/or the axis direction (sign) might be wrong. Therefore both can be altered independentty.
BistroMath is carefully designed to preserve the original data on export. Therefore axis swapping normally will not effect the export at all.

Preserve original data on export
This option in the Measurement menu regulates the effects of axis swapping on export. Its default state is on. Switching off this option introduces serious data integrity problems because swapped data will be interpreted as original, unswapped data when opened again from file. This can undermine institution wide conventions for data interpretation. Therefore the state of this menu item is always on at startup and is not taken from any configuration file. Moreover the item is locked at program start.


The direction of the data representation on the screen may be inverted when comparing side by side with the measuring system. BistroMath by default uses the standard directions from G / A / U / L to T / B / D / R for Inline, crossline, beam and diagonal scans respectively.


The GT / AB / UD / LR designations can be swapped as viewing options and is reflected in the bottom axis title color. Note that this will not influence the graph itself nor the subsequent analysis.


The axis sign (direction) can be inverted at read time as a measurement option and is reflected in the bottom axis labels color. This swaps all data from left to right and vice versa and has effect on all relevant analysis results but is not reflected by any color in the results panel. Colors in the results panel have other meanings.
