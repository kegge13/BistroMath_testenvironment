History v2.00 - v2.69


version 2.69 17/08/2014
  • Improvements on the generic profile to PDD option for completely untyped two-column data. For the related FreeScans, typed data not representing any standard scan like Inline/Crossline/Depth dose/Fanline/Diagonal, this was fully covered already.
  • In the Fit Results tab, unchecking the Include labels box now also leaves out the name of the fit.
  • Minor bug: in the Files tab not the full path name was passed, failing to open a selected file. Also unrelated information from the Editor window was used.
  • Minor bug: in the Files tab an incorrect scantype could be set. As solution the axis (re)mapping (Alternative axis setup) is set explicitely in this tab.

    version 2.68 31/07/2014
  • Measurement comments can now be displayed optionally in the axis title.
  • The minimum zoom range has been changed from 1.01 to 1.011. This forces the 101% and 99% level always to be visible on the vertical axis.
  • For unzoomed views the <arrow-up>/<arrow-down> now work on the right axis.
  • Further improvements on unintentional file locking for writing files.
  • For supported arrays (at this moment only the StarCheck Maxi) the temporary reference can also be a multi-scan file. 
  • Bugfix: With some combinations of configured preferences hotkeys in the Reference menu got lost at startup.

    version 2.67 18/07/2014
  • Bugfix: In the penumbra evaluation engine valid results were not recalculated when the normalisation value changed. The effect is only significant when the normalisation value differs significantly from the maximum value or initial normalisation value. This might be the case with double-wedge profiles where the 50% width is critical, the slopes are less steep, and a higher maximum at (possible) lobes is not used for normalisation. Also, do not forget to switch off Edge Detection fallback as for these profiles the steepest slope position might significantly differ from the 50% postion.
  • Bugfix: For MCC files correct interpretation of the positions for diagonal scans would sometimes fail, leading to a zero scan length.
  • MCC files tend to represent absolute dose values in stead of relative dose values, thus leading to very small numbers (several orders of magnitude below 1). This messed up assumptions that can be made for relative profiles. Therefore, at conversion to the internal used data format, dose values are scaled to 100 when the maximu is below 50.
  • The annotations c,e and nn, where nn is a number, are added to positional results for profile centering, edge detection and user dose level respectively. See also the results panel.
  • The annotation (s) is introduced for symmetry and flatness when the measurement is symmetry corrected.

    version 2.66 12/06/2014
    The fit models for depth dose curves use more parameters now but can be limited to the original models also. The related documentation is expanded significantly.

    version 2.65 11/06/2014
  • Bugfix: True diagonal scans that were stored as reference became 'point-to-point' scans and were not recognised anymore as diagonal. This is repaired by saving these files as 'angle' scan. Also the detection diagonal scans is extended to overcome poor registration anyway.
  • The option [View]Bottom axis always black now also keeps the bottom axis label black;
  • The option [File]Load current data, which is equivalent to tab changing, now handles the OmniPro v6 binary (*.rfb) correctly. In previous versions only the editor was reread, which effectively disconnected the binary file.
  • The file locking problem (see v. 2.63) is now avoided by using alternative mechanisms for writing.
  • A Fit Results Tab is added to export pdd-fit results in a for spreadsheet acceptable format.

    version 2.64 05/06/2014
  • Bugfix: partially scanned profiles with very limited information (typically aborted scans, "garbage") could lead to calculation errors in the matching module and left the program in a locked state.

    version 2.63 27/05/2014
  • The temporary file is now written with a unique file name. On closing BistroMath will (try to) delete this file. This avoids multi user conflicts and possible file locking on a network drive.
  • The Linac Symmetry Error is only calculated for fields with left and right border values of opposite sign.

    version 2.62 20/05/2014
  • For extrapolation of photon data a second photon fit model is added to fit the data beyond d50. An overview of the results shows that the photon extrapolation model performs much better beyond the available data range.
  • When the coloured name/date label in the Results panel is clicked, this will select the correspondig curve in the graph area will be selected.
  • Support for the help in the 32-bit ".hlp" format is dropped. This was the standard on Windows XP but Bistromath could also work with htmlhelp on that platform. Now only the htmlhelp ".chm" format is available.
  • Bugfix: with a default installation on startup the path C:\ProgramData\BistroMath\References (or C:\Documents and Settings\All Users\Application Data\BistroMath\References) was not found due to a missing backslash in the path. Without explicit reference path in the configuration file this would lead to the conclusion that the reference directory is in the program directory. However other choices are made during installation. Please check your configuration file to include RefDir=C:\ProgramData\BistroMath\references\ in the [AnalyseForm] section. If there is no file with the name bistromath.ini or institute.ini in the directory C:\ProgramData\BistroMath then make one by saving a configuration.
  • In the selftest there is now a test on loading reference files (see bugfix above). If this test is not passed this is considered "fatal" for the selftest.
  • The zoom function (key Z) now also works on the right axis of the graph.
  • The Reference directory can now be set from within the program.

    version 2.61 09/05/2014
  • Bug: With the dropping of the multi-language support in version 2.59, also the labels of the curvee information lines ("Measurement:", "Calculated:","Reference:","Buffer:") became empty.
  • Minor bug: The View menu was kept disabled after tab switching combined with Advanced mode turned off. 
  • Minor bug: The combination of the settings [Measurement]Move profiles to origin, [Reference]Align with measurement, and [Reference]Calculation is Gamma analysis did not move the reference. The profile in the file selftest02_real.txt is moved by 2 cm to demonstrate and test this behavior.
  • The Editor can now be hidden through an option in the View menu.

    version 2.22 16/02/2013
    The recently introduced axis swapping with both axis inversion in the graph area and swapping of direction indicators did not work as expected. In a readme file on the installation of OmniPro 7.4 some background information can be found. Version 7.4 complies with IEC1217 conventions while OmniPro v6 was based on IEC601. The main difference is the inline direction, which is from T to G in v7 and from G to T in v6. For now Bistromath uses from G to T direction to set up inline data. To get consistent analysis results not only the graphical representation but also the underlying data should be inverted. How this is handled in Bistromath is described in Axis swapping.
    Validation of watertank PTW data is still need badly. See data validation.

This is mentioned in a readme file on the installation disk and is related to IEC1217. This was tested with a wedge field. The data can be rewritten to the OmniPro_v6 format with correct axis direction representation. Also a measurement time with one digit for the hours did not pass the filter. The workaround was: do not measure before 10 o'clock...

    version 2.20 08/01/2013
    The PTW data format can now be written also with the limitation that only single profile files are produced. Writing into a multi data file will only be developed when there is a broadly expressed need for this feature.
    Comparing different file types and/or different tank setups might lead to a mixup of directions and would lead to significant larger complexity of the code. Therefore the data point reading code for all file types was reviewed and partly rewritten in such a way that the data are stored internally always as G»T / A»B / U»D / L»R. As the interpretation of the data itself is not changed, this review should have no effect for any previously generated data files. However newly written data files represent the internal storage conventions as decribed above.
    This reduced complexity comes at a small cost. As data are not always presented anymore in the direction as measured, this might lead to an inverted bottom axis when compared side-by-side with the measuring system, which can be corrected with the newly introduced Axis swapping.
  • The addition of date and time to the file name has been made optional. This is also reflected in the SpecialModes 2 and 3.
  • In the file conversion tab both the PTW and RFA300 were added to the supported list of file types.
  • The interpretatation of G and T side is flipped for XiO files. For AB-profiles the direction is unclear but not critical because they are symmetric. In the physics module the wedge profile is always calculated on the GT-direction, ignoring the (UMC Utrecht) setup configuration which should lead to a wedged AB-profile.
  • Bugfix: the normalisation on the average In-Field area, introduced in version 2.14, resulted in an illegal value for depth dose curves. Both data and logic are corrected.
  • Bugfix: SpecialMode2 caused an endless waiting loop due to safety measures taken in version 2.11. Also, the wedge detection result was not used but merely the wedge angle, for deciding on the correct output string. 
  • Minor bug fix: Depth dose curves without buildup part (i.e. kV depth dose curves) caused a calculation error which is now handled properly.
  • For very small wedged fields, especially on larger depths, wedge detection might fail. In SpecialMode2 this is checked with the SpecialMode[2]Float[2] parameter. SelfTest04_xio.txt can be used to test this.
  • Improved information on the file name and parameter check of the reference data.
  • Some words on the update policy.
  • The detector type, when known, is written into the wtx/wda format as comments.
    Validation for PTW is still an open issue.

    version 2.14 30/10/2012
  • For comparison across different linacs there might be more pronounced differences. Also the exchange of a gun filament may introduce unexpected differences. In the classical approach all differences are shown in relation to the central normalisation point. Now the average over the In-Field area can be chosen as normalisation for either measurement, reference or both. As alternative an arbitrary value can be set for the measurement. See normalisation of profiles for an overview.
  • After symmetry correction of the reference no rematching is done, which reduces calculation time significantly.
    Validation for PTW is still an open issue.

    version 2.13 22/10/2012
  • The average step size is no longer used to calculate a dose within the range of defined calculation width. Instead the true number of points in the calculation area is taken.
  • In automatically generated file names, as used to store reference files, the field size is left-padded with a "0" to get at least two characters. Also, when applicable, the scan angle is now correctly represented in the filename.
    Validation for PTW is still an open issue.

    version 2.12 10/10/2012
  • The interpretation of Freescans is refined. When intended as profile it is handled as a generic horizontal profile, otherwise a Fanline is detected.
  • The Measurement menu option "Generic to PDD" applies now also to fanline measurements.
  • HDF files do not contain film data anymore as default.
  • When film data are found explicitly or assumed implicitly, the file format "generic profile" is assumed for any OD to dose conversion and/or background correction.
  • For film data the edge detection based field limits are used as alternative for 50% values. Edge Detection fallback needs to be activated for this behavior.
    Validation for PTW is still an open issue.

    version 2.11 02/10/2012
  • Bugfix: before calculating a division of Measurement and Reference both curves are filtered. Unintentionally the unfiltered curves were used for this calculation. This increased the noise on the "division" curve. Therefore also the relative flatness was overestimated due to noise.
  • Where possible and meaningful, multithreading is applied. Because a lot of calculations are dependent, the gain in speed is limited.
  • Matching may be chosen as preferred method for aligning measurement and reference.
    Validation for PTW is still an open issue.

    version 2.10 26/09/2012
  • Yet another variant of the OmniPro-Accept ascii format was found: the v7.4 differs slightly from previous versions.
  • The normalisation dose and distance in the Gamma analysis are allowed to be zero. A zero setting will ignore that dimension.
  • Support for fanline scans is improved. A correct pathlength is calculated from the position data.
  • The interpretation of the Measurement menu option "Generic to PDD" is extended to Freescan/Point-To-Point profiles.
  • The OmniPro format RFA300 can now be written. These files can be imported into OmniPro.
  • The usage of the alias list is slightly changed for the linac name. It is no longer aplied on the linac name itself but only on the filename of the reference file. This preserves the name of the linac in both the measurement and the reference in the screen representation.
    Validation for PTW is still an open issue.

    version 2.09 08/08/2012
    This version has been published with a lot of subversions (in the build number range 231 - 253) over several months and is still in beta stage for mcc-data, Sun Nuclear data, improved filtering, median filtering and gamma analysis.
  • When the scaling of data of the Calculated or Buffer plot differs significantly, the right axis will be used for those curves.
  • When the buffer curve is activated and there are no data available, the derivative is presented.
  • Context-sensitive help is now given with F1, this was general help.
  • A warnings mechanism has been added to inform on non-fatal problems.
  • The introduction of Gamma analysis offers a new measure of field consistency for non-flat flields like the Flattening Filter Free beams. The dose difference can be relative to the global normalisation dose value or the local dose.
  • The new median filter is used on the results of a division or  gamma analysis . The quadratic filter is extended with a fallback strategy. The number of used points for filtering is not based on the average step size anymore. Instead only points within the range defined by the filtering width are taken. 
  • The PTW CC-Export V1.9 text format (.mcc) and Sun Nuclear text format (.snctxt) can be read. The mmc-format was also tested as clipboard data in a pre-market release of the PTW 2012 software. Validation of these formats under different circumstances is still needed at this moment. Therefore these imports are [beta-status]. Support for a new subversion of the format can be handled now too.
  • Related to the PTW data range, typically with a maximum in the order of 1, this lead to a limited data resolution when exporting to the wms format. This is solved now.
  • The diagonal detection is refined.
  • The normalisation of depth dose curves is explained extensively in this help file.
  • A private patch of Sun Nuclear also delivers a clipboard format, which might or might not be published in coming versions of that software.
  • The handling of generic profiles is refined. In default mode they are handled as profiles. Through a menu item this can be forced to a PDD type.
  • Minor bug: a non-existent references path stored in the ini file did create unexpected errors. Both the references path and the source of the unexpected errors are now checked.
  • Futile bug: the absolute flatness value is rounded to integer value when larger then 10% and a relative flatness is presented to fit within a limited character space.
  • Klemhöfer was renamed to BistroMath to acknowledge the wide range of supported data formats and the new ways of looking at numbers from profile analysis.
  • The RFA300-format is supported.
  • The list of offered file types in the Open... dialog is revised and extended.
  • Bug: incomplete lines in the alias list editor fixed.
  • Updates of this help file.

    version 2.07 25/01/2012
    Some wedged fields were not analysed correctly. A histogram is used to find a central band of points to ignore in the derived function when looking for the minimum. With very high peak maximum values almost all points were collected in one single bin with default bin size, which would lead to a missing minimum. Now the number of bins is doubled until less then 25% of the points is in a single bin. The exact minimum and maximum in the derivative are now calculated using a quadratic fit.
    The help file has been expanded with a Profile analysis strategy section which gives an overview of the behavior of the program.

Bistromath forced then a Windows system error message and failed to find the reference directory. When the drive does not exist, this version will fall back on default settings. Eventually this directory will be created.
