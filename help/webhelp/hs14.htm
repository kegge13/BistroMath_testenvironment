Institute defaults configuration
options menu 
With multiple users working on possibly installations and/or user id's it can be hard to maintain a uniform standard of analysis behavior. Any user might save the standard configuration file, making it hard to establish the stable result over longer periods. Also, starting with Windows Vista, the users are not allowed to overwrite the files in the program files directory. The users versions of these files are masked by the operating system.
  • There is a Save config as... item in the Options menu. This option is locked at program start and should be unlocked with <Ctr>+<Alt>+<R>. Through this route a configuration file can be stored with any name. The name defaults to institute.ini.
  • On startup Bistromath will look for a file institute.ini in the same directory as the executable. Note that "Save config" and "Load config" use the file bistromath.ini.
  • Alternatively the shortcut to BistroMath can be changed to load any configuration file. For instance: "C:\Program Files (x86)\BistroMath\BistroMath.exe" -config path_to_any_file. Note the spaces between executable and flag "-config" and between the flag and its value. When not a fully qualified path is given it will be interpreted as relative to the path of the executable.
    Once a decent institute.ini is created this can be distributed.
