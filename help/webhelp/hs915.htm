Installation and configuration issues
915
Application
Although BistroMath can be used as a stand-alone analysis tool it is intended as extension to water phantom systems in radiotherapy and therefore should agree with viewing conventions in the water phantom system. As BistroMath has many features this might become confusing when different users use one or more installations of the software over one or more Windows platforms. Mostly an institute wide default behavior on all installtions for all users might be preferred. This page tries to address all aspects of installation and configuration.
The names of some settings in the configuration files have been changed in version 2.53. This is corrected by the setup program in bistomath.ini and institute.ini. for standard setups. When in doubt, check the settings and save them again.
It is strongly advised to use a "." as decimal separator in the regional settings of Windows.

Configuration file consistency
The installation program can convert all old parameters to the new standards in all configuration and preset files in the data directory. This feature can also be run without installing Bistromath on any other directory. Just run the installation file, select the data directory and deselect installation of BistroMath.






Setup on different Windows platforms
Starting with version 2.50 the datastructure is reorganised to conform with Windows standards. The setup program will adjust this automatically.

Windows XP
executable C:\Program Files\BistroMath
selftest data C:\Program Files\BistroMath
data  C:\Documents and Settings\All Users\ApplicationData\BistroMath
institute.ini C:\Documents and Settings\All Users\ApplicationData\BistroMath
bistromath.ini C:\Documents and Settings\All Users\ApplicationData\BistroMath
references C:\Documents and Settings\All Users\ApplicationData\BistroMath\References

Windows Vista and later
executable C:\Program Files(x86)\BistroMath
selftest data C:\Program Files(x86)\BistroMath
data  C:\ProgramData\BistroMath
institute.ini C:\ProgramData\BistroMath
bistromath.ini C:\ProgramData\BistroMath
references C:\ProgramData\BistroMath\References

Installation on network
The installation onto a network drive should be done manually.
executable any path
selftest data same path as executable
institute.ini same path as executable
bistromath.ini same path as executable as default
data  any path
references any path configured in ini file or on command line

Startup logistics
At startup Bistromath will look for institute.ini in the standard datapath. If not found bistromath.ini will be used. On a default installation Bistromath will start correct without any configuration file. Of course, settings will not be as expected.
Alternative configurations can be used on the command line: bistromath.exe -config path_to_ini_file.
The user can load bistromath.ini after program start. See Institute defaults configuration for more details.
The philosophy behind this behavior is to archive a uniform analysis outcome for all users on all platforms without losing the opportunity to store specific user settings. When a configuration file is opened, a line is written to the messages tab. Additional, a preset can be loaded to support a dedicated task.


Lost information without any configuration file
Depth dose curves might not be scaled as expected.

Command line
The command line offers opportunities to use alternative configurations. See command line parameters for details.

Interpetation of Depth dose curves
Both analysis values as graphical representation are highly configurable and most problably should be set to institute wide preferences. See Depth dose interpretation for details.

Various settings

The most settings are described in the Settings tab.

Graph view configuration: axis swapping

For most users it is prefered that the graph view matches with the profiles as seen in the watertank software. Notably the GT-axis is handled inconsitent by different vendors or even different versions of one vendor. Therefore BistroMath offers Axis swapping.
