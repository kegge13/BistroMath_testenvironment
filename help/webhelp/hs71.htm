Set as Temporary reference
processing menu
 
When a measurement is loaded this may be used as temporary comparison to all next profiles as long as this item is checked. The scan is saved in memory. With this option the current data source (clipboard or file) is reread. All reference functionality is applied to this file. This option is not saved in the configuration file and defaults to "unchecked".

The temporary reference can be unset with the Clear Temporary Reference item (<Ctrl>+<Shift>+<T>).

Note that the temporary reference supersedes any automatically detected reference. The temporary reference is passed to the next element in the history list when you cycle through the history. Whether or not a reference is 'acceptable' depends on the circumstances. The actual mechanism works through generating curve names which is influenced by an ignore list. In the current version of BistroMath there is no option to acces this list. In the Messages tab more details can be found when a reference is not applied. Increasing the log level can give even more details.
In principle only a reference which matches all parameters, including field size, scan depth, of the measurement will be applied. This can be relieved by:
  • Ignoring the name of the Treatment Unit.
  • Not checking the Temporary Scan type at all. In this case anything goes, even comparing a profile with a depth dose.
  • Also an alias can be set, notably for the Treatment Unit.

    A temporary reference can be:
  • Any single scan.
  • A structured multi-scan file for a 2D-array that is configured in BistroMath and Device specific Reference is activated.
  • Any unorganised bunch of scans when Index multi reference is activated. In this case the first applicable scan from the generated index is taken. Building an index may take some time.
