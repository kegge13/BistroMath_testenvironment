History v1.00 - v1.99

version 1.99 25/01/2011
This version is a precursor of version 2.00 with a full selftest feature.
In this version only bugs are repaired which have been around for a indefinite time but stayed undetected due to their specific nature in hardly used features.
-parsing of floating point text-strings,
-unresponsiveness of "Accept zero steps" menu item when the same data were read,
-improved interpolation techniques for sparse data files,
-handling perfect vertical slopes en perfect horizontal profiles, see calculation details.
The usage of the OD to dose conversion parameters has been changed.

version 1.98 17/01/2011
Minor bug repair: the normalisation stategy in version 1.97 introduced a graphics-only inconsistent normalisation of reference profiles.
As default name for the "File save" menu is now made from measurement parameters and measurement time.
Resolved bug: due to a bug in a library the extension of files was not set as expected.
Dutch/English translation mixup corrected.

version 1.97 06/12/2010
An optional dual normalisation of depth dose curves is introduced to separate analysis and graphical representation. See configuration for details.
It is backward compatible with configuration files of previous versions.

version 1.96 29/10/2010
normalisation of depth dose curves can now be done at a specific depth and normalisation value. See configuration for details.
It is made backward compatible with configuration files of previous versions.
The Alias tab now has a pop-up menu.

version 1.95 13/09/2010
A faster memory management utility with memory leak detection is introduced. No leaks were found.
Resampling of data to a fixed grid is now supported. See Settings tab.
The majority of the menu items which can be (un)checked now report their status in the message bar.
The naming convention for reference files has been changed. The previous naming convention is still supported for backward compatibility.

version 1.94 19/03/2010
For SpecialMode2 the construct {constant}*x|y{[-|+]value} is supported.
Also wedge information added. See SpecialModes.

version 1.93 09/03/2010
SpecialMode2 capabilities expanded: now for any y value the (right side only for pdd) x position is evaluated. In previous versions this was limited to 20, 50, 80, 90 en dUser.
Small speed optimizations were introduced.
Minor viewing bug repair: For pdd's the bottom axis minimum value sometimes was not reset when before a profile was displayed. To get a full window version again the unzoom or zoom command could be used. Now the bottomacxis is always set to the obeserved minimum and maximum position values.

version 1.92 01/03/2010
For noisy profiles with a low resolution the search strategy from center to outside for the penumbra values might end at the wrong calculation point because of the low slope in the beginning. Therefore now the search strategy is inverted: from outside to inside, which maximises the slope. Moreover a fallback to a linear fit is used when a quadratic fit might give extreme results. The number of  points to calculate a dose in any curve using a second order fit is now configurable.
Small speed optimizations were introduced.

version 1.91 11/02/2010
The Wellhöfer scantype "Angle", used for diagonal profiles, is supported. However, when tested there where files found with missing axis definitions for the water tank. Therefore an extra entry is made in the [WellhoferObject] section of the ini-file to have default values: ICD=-YX-Z.
More information is written to the status bar.
Bug repair: due to changes in a base library the SpecialMode2 strings could not be handled.
The relative flatness is no longer displayed witin brackets but as a second value after a "/".
The error tolerance for inconsistent data is improved.

version 1.90 01/02/2010
Files produced by CMS XiO are now supported. 
In SpecialMode3 the scan depth of a profile is inserted into the file name.
The upper and lower level definitions for both photons and electrons can now be adjusted in the settings tab as a percentage with a arbitrary value. In previous versions these values could only be changed in the configuration file. where they were stored as percentage. As (undescribed) design limitation these values were exclusively limited to the set values for d90, d80, d50 and d20. In this version these values are stored as fractions, but for backward compatibility values higher than 1 are interpereted as percentage.
For reference profiles the scan depth is now included in the file name to accommodate a wider range of references. When a file exists with the old name and no new one, the old file is renamed automatically.

version 1.80 07/12/2009
In SpecialMode2 the scan depth of a profile and both x- and y-size are given.
A temporary reference is only used for curves of the same scan type.

version 1.79 11/11/2009
The concepts of SpecialMode2 are changed.
Bug repairs:
 In SpecialMode2 string#2 in stead of string#3 was used for pdd's.
 With the "File/Save as" menu item data were always written as Wellhöfer ascii.

version 1.78 29/07/2009
An earlier revision of the shift-function introduced a bug.

version 1.77 23/06/2009
Now a derivative is calculated tot determine the field limits of a wedge field and therefore the effective field. 

version 1.76 01/03/2009
In one installation the penumbra-settings were not correctly deduced from the configuration file. Decent values are forced here.

version 1.75 26/01/2009
"SpecialMode"-options are introduced. See the Special modes section.

version 1.74 03/11/2008
Minor bug: after swichting back from Settings tab to Analysis tab, the profile was reread with all menu's still in a blocked state. Therefore the current profile was not divided again by the reference profile. Now the menu's are enabled in due time.

version 1.73 04/04/2008
Bistromath does not accept scans with a zero step. Now this can be overruled with a menu item.

version 1.72 26/03/2008
Bug repair for version 1.7: when the missing penumbra option was switched on the analysis of electron depth dose curves went wrong. This option is now ignored for depth dose curves.
Minor bug repair for all previous versions: for non-wellhöfer data the filename of the imported file was not transfered.
Display issue: the plotscaling for the measured curve was also used to scale the filtered curve. This guarantees an optimal aligment of both plots. However, the normalisation point in the graph may be slightly different from the actual value. Therefore, now the filtered curve is scaled with the normalisation point.
Now extra information may by put on the clipboard (see configuration). This was made to produce data for the beam check.

version 1.71 12/03/2008
Bug repairs in the conversion tab:
-the output type selection was messed up
-the filepath edits did not respond to keys used as shortcut in the View menu.
Also the result reporting in the conversion tab is extended.

version 1.7 03/03/2008
The analysis is revised to support diagonal profiles and profiles with missing penumbra's. See the calculation details. This is now also handled correctly for reference files.

version 1.61 26/02/2008
Support for Schuster profiles is added.
For stability more error-handling was added.
Bug repair: symmetry values were calculated before an optional symmetry correction was done.

version 1.60 15/02/2008
A histogram page is added.
The minimum in the In-Field area is displayed when applicable.

version 1.5.10 07/02/2008
Bugs repaired:
-due to duplication of variable names the file name was not transferred in the file conversion tab (occurred only in version 1.59)
-a binairy file could not be selected as reference
-export to wms: mixup in coördinates
-export from wms to wellhöfer format
With the introduction of a temporary reference it is possible to use references of other linacs. As in the current concept the reference is added tot the measurement data, a lot of information is shared, which possibly no longer is identical. For now this is repaired partially. In the next major version each curve will have its own information data.
Improvements:
-profiles are filtered before division now,
-the information on parsing errors is more accurate.
Added functionality:
-all profiles can be erased with the View/Clear menu item,
-normalisation ont the field centre can now be enforced.

version 1.59 28/01/2008
Now a measurement can be used as temporary reference.

version 1.58 30/11/2007
Every curve has now its own position information. This leads to an improved positional accuracy when two curves are divided.
A curve can be selected, which, in this version, has only visual effects.

version 1.57 22/11/2007
Wellhöferformat-fix: support for 'Point To Point scans'.
Bug-fix in Filesave-dialog: the filename of the input was actually used. 

version 1.56 01/11/2007
Generic-format-fix: read error on two columns.

version 1.55 25/10/2007
WDA-format-fix: wda-files produced by recent Plato versions seem to produce a slightly different header. This is also accepted now.

version 1.54 18/10/2007
Position analysis at user dose levels can be toggled on/off.
The help file is extended.
When the mouse is clicked in the graph window, the cursor is moved to that position.
Bugfix: the vertical alignment of measurement and reference profiles could go wrong due to wrong normalisation values.

version 1.51 09/10/2007
Position analysis at user dose levels is introduced.

version 1.50 09/09/2007
OD to dose conversion is added.

version 1.41 03/09/2007
Support for Pips profiles was added.
The Editor window is set to Plain Text and no Word wrap explicitely before every read action.

version 1.40 31/08/2007
Support for generic profiles and hdf profiles was added. For selected file types a background may be subtracted.
The configuration will only be rewriiten at program exit if an autosave option is set.
Several minor bugs were repaired.
The clipboard may be ignored through a new item in the File menu.

version 1.35 10/12/2006
The depth of dose maximum is given for pdd-curves.

version 1.34 27/11/2006
Minor bug: when a file is read from the references directory it cannot be saved. An appropriate message is produced.

version 1.33 22/11/2006
More efficient use of resources.
Overwite file checkbox for file conversion.
Added "Extra" curve for a new facility in future.

version 1.32 14/11/2006
Introduction of file name generation.
Bug repaired in language library load procedure. Possible source of strange errors at startup in previous versions.

version 1.31 13/11/2006
Addition of "force lower case names" on conversion page.
Addition of "Maximum within flatttened area" for profiles on analysis page.

version 1.30 06/11/2006
Introduction of a multi-tabbed layout.
The wms-formats (*.wtx and *.wda) are supported in this version for import and export. On a separate page groups of files can be converted. Aliases are introduced. 

version 1.22 16/08/2006
Minor bug fix.

version 1.21 12/07/2006
The filter width can be accessed with a menu option.

version 1.20 05/05/2006
Bug fix: the penumbra-high value was read from the low value, therefore the "20-20" value was calculated (correctly) as zero.

version 1.19 18/03/2006
The penumbra definition is now split up in a electron value (default 90-50) and a photon value (default 80-20). Both have there own entry in the ini file and  will be used when appropriate.

version 1.18 18/03/2006
Bottom axis title in larger and bold format.

version 1.17 08/03/2006
For Measurement and Reference file, also the name of registrered linac is displayed.

version 1.16 28/02/2006
Bug repair: When there was no information on the reference directory, there was an incorrect default.

version 1.15 02/02/2006
Bug repair: An error in the reading of the filter value could lead to zero output of the filter.

version 1.14 22/12/2005
Film-scanner files are now supported.
The filter width is made available within the program.

version 1.13 19/12/2005
For very narrow profiles the display-normalisation is improved.

version 1.12 18/12/2005
The BadPenumbra-limit is added to the configuration file. A menu item "Accept bad penumbra's"is added.

version 1.11 14/12/2005
The internal notation of the energy is changed to a floating type to accomodate kV-range energies. When the energy value (in MV) is less then 1, this value is presented as kilovolts.

version 1.10 02/12/2005
The color scheme is changed to a white background of the graph.
The value of any position in the curve can be displayed (values are interpolated as needed).
Profiles may be centered to make the 50%-point symmetric.
Profiles can be scaled to the surface-distance.
The total width of profiles is given.
